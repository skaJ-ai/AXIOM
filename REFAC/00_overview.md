# AXIOM 총평

## 한 줄 정의
AXIOM은 **AI에게 질문하는 앱**이 아니라,
**HR 업무를 구조화하고 중간 산출물을 축적하는 work OS / copilot** 쪽으로 가고 있는 제품입니다.

## 핵심 베팅
이 제품의 핵심은 “더 좋은 답변”이 아니라 다음에 있습니다.
- 4개의 사고 모드로 작업 분해
- 중간 artifact를 1급 객체로 저장
- deliverable 중심 workflow
- memory/retrieval을 통한 재사용

이 방향은 generic AI chat 앱보다 훨씬 낫습니다.

## 현재 상태 판단
### 실제로 강한 것
- 4-mode thinking architecture
- write mode 기반 session canvas UX
- intermediate artifact 보존 구조
- deliverable lifecycle (draft/final/promoted)
- hybrid retrieval과 knowledge/memory 뼈대

### 아직 약한 것
- structured output contract의 안정성
- evidence/provenance의 사용자 노출 수준
- connector/action layer 실체
- governance/shared workspace 현실화
- product narrative 대비 구현 정합성

## 최종 판정
AXIOM은 허세형 AI SaaS가 아닙니다.
실제로 제품 철학과 구조가 있고, 코어도 어느 정도 구현되어 있습니다.
다만 지금은
**좋은 4-mode AI workbench**에 가깝고,
문서/위키/README에서 약속하는 더 큰 플랫폼 비전은 아직 부분 구현 상태입니다.
